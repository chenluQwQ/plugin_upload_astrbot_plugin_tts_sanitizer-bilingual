# TTS文本过滤插件

一个智能的AstrBot TTS文本过滤插件，通过透明包装 TTS Provider 来过滤不适合语音朗读的内容，不修改消息链，不产生额外消息。

## 已重构！使用了新的处理流程，现在又可以正常使用了！

## ✨ 主要特性

- **� 透明过滤**: 直接在 TTS Provider 层面拦截，消息链完全不受影响
- **� 颜文字过滤**: 自动移除 `(＾_＾)`, `(>_<)`, `o_o` 等颜文字
- **🎨 特殊符号清理**: 过滤 ★♪♥💖→ 等装饰性符号和表情符号
- **🔄 智能替换**: `233` → `哈哈哈`, `666` → `厉害`, `555` → `呜呜呜`

## 🚀 快速开始

### 安装方法
1. 将插件文件放入AstrBot的插件目录
2. 重启AstrBot或使用插件管理功能加载
3. 插件会自动启用，使用默认配置即可开始工作

### 基本使用
插件安装后会自动工作，无需额外配置。用户收到的消息与未安装插件时**完全一致**，只是音频内容经过了过滤：

- 🔊 **音频**：朗读过滤后的干净内容（无颜文字、无怪读音）
- 💬 **文本**（如启用 dual_output）：显示**原始完整内容**，保留所有颜文字和表情

> 💡 **与 v0.x 版本的区别**: v1.0 不再修改消息链，不再发送额外消息。无论 `dual_output` 设为 True 还是 False，显示的文本永远是原始的。

## ⚙️ 配置说明

插件支持通过AstrBot配置面板进行设置，主要配置项包括：

### 基础设置（3项）
- **启用过滤** (`enabled`): 总开关
- **最大朗读字数** (`max_length`): 超过此字数直接跳过朗读，防止 token 浪费（0=无限制）
- **最大处理长度** (`max_processing_length`): 超过此长度跳过正则过滤，防止卡顿

### 过滤规则（3项）
- **正则过滤** (`remove_patterns`): 正则表达式列表，匹配到的内容会被删除（括号内容、颜文字、装饰符号等）
- **直接过滤** (`filter_words`): 会被 TTS 读出怪声的字符和词汇（希腊字母、俄文字母、网络用语等）
- **替换词汇** (`replacement_words`): 格式为"原词|替换词"，将网络用语转为可朗读的词

### 其他（2项）
- **重复压缩** (`max_repeat_count`): 保留的最大重复数量，0=关闭（如设为2，哈哈哈哈→哈哈）
- **调试模式** (`debug_mode`): 在日志中显示每次过滤的详细对比

## 🔧 工作原理

### 核心机制：透明包装 TTS Provider

```
┌─────────────┐
│  LLM 回复   │
└──────┬──────┘
       │
       ▼
┌──────────────────────────────┐
│  AstrBot 结果装饰阶段         │
│  (消息链保持原始文本不变)      │
└──────┬───────────────────────┘
       │
       ▼
┌──────────────────────────────┐
│  AstrBot 调用 TTS Provider   │
│  get_audio("原始文本")        │
└──────┬───────────────────────┘
       │
       ▼
┌──────────────────────────────┐
│  ★ 本插件的包装层拦截         │
│  filter_text("原始文本")      │
│  → 调用原始 get_audio("过滤后")│
└──────┬───────────────────────┘
       │
       ▼
┌──────────────────────────────┐
│  用户收到：                    │
│  🔊 音频 = 过滤后的朗读       │
│  💬 文本 = 原始完整内容       │
└──────────────────────────────┘
```

**关键优势**: 消息链从未被修改，不产生任何额外消息。

### 过滤流程
应用以下过滤规则：
1. **颜文字移除** - 使用正则表达式匹配并删除
2. **直接过滤词汇移除** - 完全删除不适合朗读的词
3. **词汇替换** - 将网络用语转换为可朗读的词汇
4. **特殊符号移除** - 删除装饰性符号和表情
5. **重复字符压缩** - 将多个重复字符压缩到指定数量
6. **多余空格清理** - 清理多余的空白字符

### 技术说明

**为什么采用 Provider 包装？**
- v0.x 版本通过修改消息链来过滤 TTS 文本，但会导致显示文本也被修改
- 不得不通过 `after_message_sent` 补发原始文本，体验割裂
- v1.0 直接在 TTS Provider 的 `get_audio()` 入口拦截，从根源解决问题
- 消息链完全不受影响，`dual_output` 设为 True 或 False 都能正常工作

**Live Mode 支持**
- 对支持流式的 TTS Provider，同时包装 `get_audio_stream()` 方法
- 在文本队列和 Provider 之间插入过滤中间层

**干净卸载**
- 插件卸载时自动恢复所有 Provider 的原始方法
- 不会留下任何残留

## 🐛 故障排除

### 常见问题

**Q: 过滤不生效怎么办？**
A: 使用 `/tts_filter_stats` 检查"已包装 Provider"数量。如果为 0，说明插件加载时没有 TTS Provider，尝试 `/tts_filter_reload`。

**Q: 配置修改后不生效？**
A: 使用 `/tts_filter_reload` 重新加载配置，或重启AstrBot。

**Q: 正则表达式报错？**
A: 检查配置中的正则表达式语法，可以使用在线正则测试工具验证。

**Q: TTS 朗读超长文本导致卡顿？**
A: 调整 `max_processing_length` 配置，对超长文本跳过处理。

### 调试方法
1. **开启调试模式**: 在配置中设置 `debug_mode: true`
2. **使用测试命令**: `/tts_filter_test` 验证过滤效果
3. **检查统计信息**: `/tts_filter_stats` 确认 Provider 包装状态
4. **查看日志**: 开启调试模式后查看详细的过滤过程
